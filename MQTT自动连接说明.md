# MQTT自动连接和重连功能说明

## 概述

本次更新为MQTT客户端添加了以下功能：
1. ✅ **应用启动时自动连接MQTT**
2. ✅ **连接断开时自动重连**
3. ✅ **详细的连接日志**
4. ✅ **优雅的停止机制**

---

## 功能特性

### 1. 自动启动连接

**通过 FastAPI 启动事件实现**

无论您使用哪种方式启动程序，MQTT客户端都会自动启动：

```bash
# 方式1：通过 app.py 启动（推荐）
python app.py

# 方式2：直接运行 web_app.py
python web_app.py
```

**启动流程：**
```
应用启动
  ↓
检查/创建配置文件
  ↓
初始化配置管理器
  ↓
自动启动MQTT客户端
  ↓
尝试连接MQTT服务器
  ↓
连接成功 → 开始接收消息
```

---

### 2. 自动重连机制

**特性：**
- ✅ 意外断开时自动尝试重连
- ✅ 使用指数退避策略（避免频繁重连）
- ✅ 重连延迟：5秒 → 7.5秒 → 11.25秒 → ... → 最大60秒
- ✅ 无限次重试，直到连接成功
- ✅ 连接成功后重置重连计数器

**重连触发条件：**
- 网络断开
- MQTT服务器崩溃/重启
- 连接超时
- 任何意外断开（错误码 rc != 0）

**不会触发重连：**
- 主动调用 `stop()` 方法
- 应用关闭
- 正常断开连接

**重连日志示例：**
```

  重连次数: 1
  延迟时间: 5.0 秒
======================================================================

开始重连到MQTT服务器...

  返回码 (rc): 0
  标志 (flags): {'session present': 0}
  ✓ 连接成功！
  ✓ 重连成功！（共尝试 1 次）
```

## 配置参数

在 `config/printer_config.json` 中配置MQTT连接：

```json
{
  "mqtt": {
    "url": "ws://10.100.10.121:8083/mqtt",
    "topic": "zebra/print",
    "username": "admin",
    "password": null,
    "client_id": null
  }
}
```

**支持的URL格式：**
- `mqtt://host:1883` - 标准MQTT（TCP）
- `ws://host:8083/mqtt` - WebSocket
- `wss://host:443/mqtt` - WebSocket Secure（SSL）
- `mqtts://host:8883` - MQTT over SSL/TLS

## 测试方法

### 测试自动启动

1. 启动程序：
   ```bash
   python app.py
   ```

2. 观察日志输出，应该看到：
   - "应用启动事件 - 自动启动MQTT客户端"
   - 完整的MQTT连接过程日志
   - "✓ MQTT服务已启动，等待打印任务..."

### 测试自动重连

1. 启动程序并等待连接成功

2. 停止MQTT服务器（模拟网络断开）

3. 观察日志，应该看到：
   ```
   [回调] on_disconnect 被触发 - 错误码: X
   ✗ 意外断开连接，错误码: X
   
   ======================================================================
   自动重连机制已启动
     重连次数: 1
     延迟时间: 5.0 秒
   ======================================================================
   ```

4. 等待5秒，观察第一次重连尝试

5. 重启MQTT服务器

6. 观察日志，应该看到：
   ```
   [回调] on_connect 被触发
     ✓ 连接成功！
     ✓ 重连成功！（共尝试 X 次）
   ```

## 故障排查

### 问题1：程序启动但没有看到MQTT连接日志

**检查项：**
1. 确认使用的是 `app.py` 启动
2. 检查是否有错误信息
3. 查看日志文件：`data/logs/mqtt_client_YYYYMMDD.log`

### 问题2：连接失败

**常见原因：**
1. MQTT服务器地址或端口错误
2. 网络不通
3. 防火墙阻止连接
4. 认证信息错误

**排查步骤：**
1. 检查配置文件 `config/printer_config.json`
2. 测试网络连通性：`ping 服务器地址`
3. 查看详细日志了解具体错误
4. 检查MQTT服务器日志

### 问题3：断开后不自动重连

**可能原因：**
1. 是正常断开（不会触发重连）
2. 程序正在停止中
3. 查看日志确认是否有重连尝试

## 技术实现

### 关键代码位置

1. **启动事件**：`web_app.py` - FastAPI startup事件
2. **MQTT客户端**：`src/core/mqtt_client.py`
3. **自动重连**：`mqtt_client.py` - `on_disconnect()` 回调
4. **停止方法**：`mqtt_client.py` - `stop()` 方法

### 线程模型

```
主线程（Web服务）
  ↓
启动事件触发
  ↓
创建MQTT后台线程 ────→ MQTT事件循环线程
  ↓                      ↓
继续处理Web请求         接收MQTT消息
  ↓                      ↓
                      检测到断开
                         ↓
                   启动重连线程 ────→ 延迟后尝试重连
                         ↓               ↓
                      继续循环         连接成功
```
