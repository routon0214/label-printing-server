================================================================
                    乱码问题修复说明
================================================================

【问题】
Windows控制台中日志显示为乱码，例如：
  鉁?宸茶繛鎺ュ埌MQTT鏈嶅姟鍣?
  鉁?璁㈤槄鎴愬姛

【原因】
代码中使用了Unicode特殊字符（✓ ✗ ⚠），Windows默认编码无法正确显示

【已修复】
✅ 替换了31个文件中的所有Unicode特殊字符
✅ 创建了UTF-8编码查看工具
✅ 创建了启动脚本

================================================================
                    常见问题
================================================================

【Q1: 为什么使用 [OK] 而不是 ✓？】
A: [OK] 是ASCII字符，在所有环境中都能正确显示
   ✓ 是Unicode字符，在Windows默认编码中会显示为乱码

【Q2: 旧日志文件中仍有乱码？】
A: 旧日志文件包含旧的Unicode字符
   使用 "查看日志.bat" 可以正确显示
   新生成的日志不会再有这些字符

【Q3: 如何完全避免乱码？】
A: 
   1. 使用提供的工具（查看日志.bat、start_utf8.bat）
   2. 所有新代码避免使用Unicode特殊字符
   3. 遵循项目的编码规范

【Q4: 打印输出还是乱码？】
A: 打印乱码是另一个问题：
   - ZPL标签: 检查字体和Pillow库
   - ESC/POS: 已自动使用GB2312编码
   - 运行诊断: python scripts/diagnose_encoding.py

【Q5: 需要重新构建exe吗？】
A: 如果使用exe版本，需要重新构建：
   python scripts/build.py
