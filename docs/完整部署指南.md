# 完整部署指南

## 🎯 从安装到运行的完整流程

### 第一步：检查系统环境

```bash
# Windows
.\诊断和修复.bat

# Linux
chmod +x 诊断和修复.sh
./诊断和修复.sh
```

这个诊断工具会自动：
- ✅ 检查 Python 版本
- ✅ 检查项目文件完整性
- ✅ 创建必要的目录
- ✅ 测试依赖包
- ✅ 给出具体的修复建议

### 第二步：安装依赖

如果诊断工具提示缺少依赖：

```bash
pip install -r requirements.txt
```

### 第三步：配置打印机

编辑 `config/printer_config.json`：

```json
{
  "mqtt": {
    "host": "127.0.0.1",
    "port": 1883,
    "topic": "zebra/print"
  },
  "printer": {
    "name": "ZT411"
  }
}
```

💡 打印机名称支持模糊匹配，不需要完整名称

### 第四步：启动服务

**开发模式**:
```bash
python app.py
```

**使用启动脚本**:
```bash
# Windows
start.bat

# Linux
./start.sh
```

**编译后的可执行文件**:
```bash
# Windows
dist\windows-amd64\label-printing-server.exe

# Linux
dist/linux-x86_64/label-printing-server
```

### 第五步：测试

发送测试MQTT消息：

```bash
cd tests
python test_mqtt_send.py
```

## 🛡️ 错误处理保证

### 保证1: 不会闪退 ✅

所有错误都会：
1. 显示清晰的错误信息
2. 显示完整的错误堆栈
3. 给出具体的排查建议
4. 等待用户按回车确认

### 保证2: 自动创建目录 ✅

程序启动时自动创建：
- `config/` - 配置文件目录
- `logs/` - 日志文件目录
- `failed_labels/` - 失败任务目录

### 保证3: 编码问题自动修复 ✅

Windows 控制台 UTF-8 编码自动修复，不会出现乱码或编码错误。

### 保证4: 多层异常保护 ✅

5层异常捕获机制：
1. 控制台编码修复
2. 最外层异常捕获（导入错误）
3. main() 函数异常捕获
4. MQTT 客户端异常捕获
5. 文件保存异常捕获

## 📋 故障排查流程

### 问题：程序启动失败

**步骤1**: 运行诊断工具
```bash
.\诊断和修复.bat
```

**步骤2**: 根据诊断结果操作
- 缺少依赖 → `pip install -r requirements.txt`
- Python 版本问题 → 升级到 Python 3.7+
- 文件权限问题 → 以管理员身份运行

**步骤3**: 再次运行
```bash
python app.py
```

### 问题：找不到打印机

程序会自动列出所有可用打印机：

```
发现 2 台打印机:
  1. Microsoft Print to PDF
  2. ZDesigner ZT411-300dpi
```

在配置文件中使用：
- 完整名称: `"ZDesigner ZT411-300dpi"`
- 模糊匹配: `"ZT411"` 或 `"zebra"`

### 问题：MQTT 连接失败

**错误提示**:
```
======================================================================
错误: Connection refused
======================================================================

提示：
  1. 检查MQTT服务器是否运行
  2. 检查打印机配置是否正确
  3. 查看上方的详细错误信息

按回车键退出...
```

**解决方法**:
1. 确认 MQTT 服务器地址和端口正确
2. 检查防火墙设置
3. 测试 MQTT 服务器连接：
   ```bash
   mosquitto_sub -h 127.0.0.1 -t test
   ```

### 问题：打印失败

**现象**: 标签保存到 `failed_labels/` 目录

**排查**:
1. 检查打印机是否开机
2. 检查打印机连接（USB/网络）
3. Windows: 检查打印机驱动
4. Linux: 检查 CUPS 配置
