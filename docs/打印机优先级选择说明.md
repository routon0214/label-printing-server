# 打印机优先级选择机制说明

## 概述

新的打印机选择机制实现了**专用打印机优先，通用打印机备选**的策略，取消了"默认打印机"的概念。

## 设计原则

### 1. 专用打印机优先 ⭐
- 配置特定类型的打印机（如 `types: ["label"]`、`types: ["pdf"]`）
- 优先处理对应类型的打印任务
- 性能最优，专机专用

### 2. 通用打印机备选 🔄
- 配置通配符类型的打印机（`types: ["*"]`）
- 仅在找不到专用打印机时使用
- 一机多用，灵活配置

### 3. 无默认打印机概念
- 不再有"默认打印机"标记
- 根据实际需求智能选择
- 配置更清晰

## 配置示例

### 示例1：专用打印机 + 通用打印机（推荐）

```json
{
  "printers": [
    {
      "name": "ZT411",
      "types": ["label"],
      "ip": null,
      "_comment": "标签专用打印机"
    },
    {
      "name": "KONICA",
      "types": ["*"],
      "ip": null,
      "_comment": "通用打印机（备选）"
    }
  ]
}
```

**工作逻辑：**
- `label` 类型 → 使用 ZT411（专用）
- `pdf` 类型 → 使用 KONICA（备选）
- `receipt` 类型 → 使用 KONICA（备选）

### 示例2：只有通用打印机

```json
{
  "printers": [
    {
      "name": "KONICA",
      "types": ["*"],
      "ip": null,
      "_comment": "处理所有类型"
    }
  ]
}
```

**工作逻辑：**
- 所有类型都使用 KONICA（备选）

### 示例3：所有类型都有专用打印机

```json
{
  "printers": [
    {
      "name": "ZT411",
      "types": ["label"],
      "_comment": "标签打印"
    },
    {
      "name": "HP LaserJet",
      "types": ["pdf"],
      "_comment": "文档打印"
    },
    {
      "name": "佳博GP-58",
      "types": ["receipt"],
      "_comment": "小票打印"
    }
  ]
}
```

**工作逻辑：**
- 每种类型都使用各自的专用打印机
- 无备选打印机

## 选择逻辑

```
接收打印任务 (type: 'pdf')
         ↓
查找专用 PDF 打印机
         ↓
    找到了？
    ├─ 是 → 使用专用 PDF 打印机 ⭐
    └─ 否 ↓
         查找通用打印机
              ↓
          找到了？
          ├─ 是 → 使用通用打印机 🔄
          └─ 否 → 打印失败 ✗
```

## 技术实现

### 数据结构

```python
class LabelPrintMQTT:
    def __init__(self, ...):
        # 专用打印机映射 {type: printer_instance}
        self.printer_map = {}
        
        # 备选打印机配置 (types: ["*"])
        self.fallback_printer_config = None
        
        # 备选打印机实例缓存
        self.fallback_printers = {}
```

### 核心方法

```python
def _get_printer(self, print_type):
    """
    获取指定类型的打印机
    1. 优先返回专用打印机
    2. 找不到则创建并返回备选打印机
    3. 都没有则返回 None
    """
    # 1. 查找专用打印机
    if print_type in self.printer_map:
        return self.printer_map[print_type]
    
    # 2. 使用备选打印机
    if self.fallback_printer_config:
        if print_type not in self.fallback_printers:
            # 懒加载：首次使用时才创建实例
            self.fallback_printers[print_type] = create_printer(...)
        return self.fallback_printers[print_type]
    
    # 3. 没有找到
    return None
```

## 优势

### ✅ 性能优化
- 专用打印机直接映射，无需查找
- 备选打印机懒加载，节省资源

### ✅ 配置灵活
- 可以只配置通用打印机（简单场景）
- 可以配置多个专用打印机（复杂场景）
- 可以混合配置（推荐）

### ✅ 逻辑清晰
- 没有"默认打印机"的歧义
- 专用、备选概念明确
- 日志输出清晰

### ✅ 扩展性强
- 添加新打印类型无需修改核心逻辑
- 支持多台通用打印机（取最后一个）
- 支持打印机热插拔（重启服务即可）

## 配置冲突处理

### 规则1：专用打印机不会被覆盖

```json
{
  "printers": [
    {"name": "ZT411", "types": ["label"]},
    {"name": "KONICA", "types": ["*"]},
    {"name": "另一台", "types": ["label"]}  // ⚠ 会被跳过
  ]
}
```

**输出：**
```
✓ 标签打印（专用） → ZT411
✓ KONICA 设置为通用打印机（备选）
⚠ 标签打印机已存在，跳过 另一台
```

### 规则2：多个通用打印机，取最后一个

```json
{
  "printers": [
    {"name": "打印机A", "types": ["*"]},
    {"name": "打印机B", "types": ["*"]}  // 这个会覆盖
  ]
}
```

**最终使用：** 打印机B

### 规则3：配置顺序影响专用打印机选择

```json
{
  "printers": [
    {"name": "打印机A", "types": ["pdf"]},  // 优先
    {"name": "打印机B", "types": ["pdf"]}   // 被跳过
  ]
}
```

**建议：** 把最重要的专用打印机放在前面

## 常见问题

### Q1: 如何配置一台打印机处理所有任务？

**A:** 使用通配符 `["*"]`

```json
{"name": "KONICA", "types": ["*"]}
```

### Q2: 专用打印机离线了怎么办？

**A:** 系统会尝试使用备选打印机。建议：
1. 配置一台通用打印机作为备选
2. 监控打印失败日志
3. 及时修复专用打印机

### Q3: 可以配置多台备选打印机吗？

**A:** 目前只支持一台备选打印机（最后配置的生效）。建议使用负载均衡器或打印服务器管理多台打印机。

### Q4: 如何知道使用的是哪台打印机？

**A:** 查看打印日志：
```
类型: PDF文档打印
  使用: PDF专用打印机  ← 这里会显示
```
