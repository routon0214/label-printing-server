# 打印队列系统说明

## 功能概述

打印队列系统实现了任务队列机制，确保打印任务依次执行，避免并发打印造成的冲突和错误。

## 核心特性

### 1. 任务队列
- ✅ 所有打印任务先进入队列，依次执行
- ✅ 支持单线程处理，避免并发冲突
- ✅ FIFO（先进先出）顺序

### 2. 文件缓存
- ✅ 任务自动保存为JSON文件
- ✅ 存储目录：`data/print_queue/`
- ✅ 文件命名：`task_YYYYMMDD_HHMMSS_微秒.json`

### 3. 自动清理
- ✅ 打印完成后自动删除任务文件
- ✅ 不管成功与否都会删除
- ✅ 保持队列目录清洁

### 4. 实时监控
- ✅ Web界面显示队列状态
- ✅ 显示队列长度、已处理数、失败数
- ✅ 计算成功率

## 系统架构

### 组件结构

```
打印队列系统
├── PrintQueue (队列管理器)
│   ├── 任务接收
│   ├── 文件存储
│   ├── 队列处理
│   └── 状态统计
├── 工作线程
│   ├── 任务获取
│   ├── 打印执行
│   └── 文件清理
└── 状态监控
    ├── 队列长度
    ├── 处理统计
    └── 成功率
```

### 工作流程

```
1. 接收打印请求
   ↓
2. 标准化数据格式
   ↓
3. 保存任务到文件 (task_*.json)
   ↓
4. 加入队列
   ↓
5. 工作线程获取任务
   ↓
6. 执行打印
   ↓
7. 删除任务文件（不管成功与否）
   ↓
8. 更新统计信息
```

## 使用方法

### 自动启用

打印队列在应用启动时自动初始化和启动，无需手动配置。

```python
# 应用启动时自动执行
print_queue = get_print_queue()
print_queue.start()
```

### API调用

所有打印请求自动使用队列：

```bash
# POST请求会自动加入队列
curl -X POST http://127.0.0.1:5000/api/print/raw \
  -H "Content-Type: application/json" \
  -u admin:admin123 \
  -d '{
    "print_type": "label",
    "format": "zpl",
    "content": "^XA^FO50,50^A0N,40,40^FDTest^FS^XZ"
  }'
```

**响应示例**：

```json
{
  "success": true,
  "message": "LABEL打印任务已加入队列",
  "task_id": "task_20251106_123456_789012",
  "queue_size": 1
}
```

### Web界面

在Web界面顶部状态栏可以看到：
- **打印队列**：当前队列状态
  - `✓ 空闲` - 队列空闲
  - `⏳ N个任务` - 队列中有N个待处理任务
  - `✗ 未运行` - 队列未启动（异常情况）
  
- **已处理/失败**：统计信息
  - 格式：`已处理数/失败数 (成功率%)`
  - 例如：`10/2 (80%)` 表示处理了10个，失败2个，成功率80%

### 查询队列状态

```bash
curl http://127.0.0.1:5000/api/queue/status \
  -u admin:admin123
```

**响应**：

```json
{
  "success": true,
  "status": {
    "is_running": true,
    "queue_size": 2,
    "processed_count": 15,
    "failed_count": 1,
    "success_rate": 93.33
  }
}
```

### 并发控制

- 默认 `max_workers=1`，确保单线程执行
- 避免多个打印任务同时发送到同一台打印机
- 防止打印机接收缓冲区溢出

## 优势

### 1. 避免冲突
- ✅ 串行执行，避免并发打印冲突
- ✅ 防止打印机接收缓冲区溢出
- ✅ 减少打印错误

### 2. 可靠性
- ✅ 任务持久化到文件
- ✅ 应用崩溃可以恢复未处理任务
- ✅ 自动错误处理

### 3. 可监控
- ✅ 实时显示队列状态
- ✅ 统计成功率
- ✅ 便于问题排查

### 4. 自动清理
- ✅ 打印完成自动删除任务文件
- ✅ 不占用磁盘空间
- ✅ 保持系统整洁

## 注意事项

### 1. 任务顺序

任务严格按照加入队列的顺序执行（FIFO），不支持优先级队列。

### 2. 并发限制

默认单线程执行（`max_workers=1`），不建议修改。如需并发：
- 确保打印机支持
- 测试稳定性
- 可能需要为不同打印机创建独立队列

### 3. 任务失败

任务失败后会：
- 更新失败统计
- 删除任务文件
- 继续处理下一个任务
- **不会重试**

如需重试机制，需要手动重新提交打印任务。

### 4. 队列恢复

应用启动时不会自动加载队列目录中的遗留任务文件。如需恢复：
1. 手动读取 `data/print_queue/` 中的任务文件
2. 通过API重新提交

## 配置

### 修改队列目录

```python
# 在 src/utils/print_queue.py 中修改
print_queue = PrintQueue(
    queue_dir='custom/queue/path',  # 自定义路径
    max_workers=1
)
```

### 修改并发数

```python
# 不建议修改，可能导致打印冲突
print_queue = PrintQueue(
    queue_dir='data/print_queue',
    max_workers=2  # 允许2个并发任务
)
```

## 故障排查

### 队列未启动

**症状**：状态栏显示"✗ 未运行"

**解决**：
1. 检查控制台是否有错误日志
2. 重启应用
3. 查看 `data/print_queue/` 目录权限

### 任务积压

**症状**：队列长度持续增加

**原因**：
- 打印机离线
- 打印机响应慢
- 网络问题

**解决**：
1. 检查打印机连接
2. 查看日志中的错误信息
3. 测试打印机是否正常

### 任务文件残留

**症状**：`data/print_queue/` 中有旧的JSON文件

**原因**：
- 应用异常退出
- 任务处理中断

**解决**：
1. 手动删除旧文件
2. 或通过API重新提交
