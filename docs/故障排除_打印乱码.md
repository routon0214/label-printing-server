# 打印乱码问题排除指南

## 问题描述

打印输出显示为乱码或无法正确显示中文字符。

## 可能的原因

### 1. 文件编码问题

**症状**: 上传的文件打印后显示乱码或问号

**原因**: 
- 文件使用了GBK、GB2312或其他非UTF-8编码
- 文件包含特殊字符或不可见字符

**解决方案**:
```bash
# 方法1: 使用文本编辑器转换编码
# - 打开文件，选择"另存为"
# - 编码选择: UTF-8 (无BOM)
# - 保存并重新上传

# 方法2: 使用命令行转换（Linux/Mac）
iconv -f GBK -t UTF-8 input.txt > output.txt

# 方法3: 使用Python转换
python -c "open('output.txt', 'w', encoding='utf-8').write(open('input.txt', encoding='gbk').read())"
```

### 2. 缺少中文字体

**症状**: 中文字符显示为方块或乱码

**原因**: 
- 系统没有安装中文字体
- 字体文件路径不正确

**解决方案**:

#### Windows系统
```cmd
# 检查字体目录
dir C:\Windows\Fonts\*yh*.* 
dir C:\Windows\Fonts\sim*.* 

# 如果缺少字体，从另一台Windows机器复制：
# - 微软雅黑: msyh.ttc
# - 黑体: simhei.ttf
# - 宋体: simsun.ttc
```

#### Linux系统
```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install fonts-wqy-zenhei fonts-wqy-microhei

# CentOS/RHEL
sudo yum install wqy-zenhei-fonts wqy-microhei-fonts

# 验证字体安装
fc-list :lang=zh
```

### 3. Pillow库未安装或版本过低

**症状**: 控制台显示"需要安装Pillow库"

**解决方案**:
```bash
# 安装Pillow
pip install Pillow

# 或更新到最新版本
pip install --upgrade Pillow

# 验证安装
python -c "from PIL import Image; print('Pillow OK')"
```

### 4. ZPL代码生成问题

**症状**: ZPL代码中缺少图像数据（^GFA命令）

**原因**:
- 文本未被识别为中文
- 图像转换失败

**解决方案**:
- 查看控制台日志，确认文本转换过程
- 确保中文字符在Unicode范围内（\u4e00-\u9fff）

## 常见问题解答

### Q1: 为什么英文和数字正常，只有中文乱码？

**A**: 这是典型的字体问题。英文使用ZPL内置字体，中文需要转换为图像。请检查：
- Pillow是否已安装
- 中文字体是否存在
- 运行诊断工具确认

### Q2: 文件编码已经是UTF-8，为什么还是乱码？

**A**: 可能的原因：
1. 文件使用了"UTF-8 with BOM"，请改用"UTF-8 without BOM"
2. 文件中包含不可见字符或控制字符
3. 使用文本编辑器打开，检查是否有异常字符

### Q3: 网络打印正常，USB打印乱码

**A**: 可能是USB驱动或打印机设置问题：
1. 检查打印机USB驱动是否最新
2. 尝试使用网络打印（更可靠）
3. 检查打印机语言设置（应为ZPL模式）

### Q4: 之前能打印，现在突然乱码

**A**: 检查最近的变更：
1. 系统更新可能删除了字体文件
2. Python库被更新或重装
3. 配置文件被修改
4. 运行诊断工具重新检查环境

## 技术说明

### 中文打印原理

本系统处理中文的方式：

1. **检测中文字符**: 
   - 检查文本是否包含Unicode中文字符（\u4e00-\u9fff）

2. **文本转图像**:
   - 使用Pillow库和中文字体渲染文本
   - 生成黑白位图图像

3. **图像转ZPL**:
   - 将位图转换为十六进制数据
   - 生成ZPL图像命令（^GFA）

4. **发送到打印机**:
   - 以UTF-8编码发送ZPL命令
   - 打印机解析并打印图像
