# 日志模块使用说明

## 功能概述

新增的日志模块提供了完整的日志查看、搜索、过滤和管理功能，帮助您快速定位和解决打印问题。

## 主要特性

### 1. 自动日志保留（7天）
- ✅ **自动清理**：系统每天凌晨3点自动清理7天前的日志文件
- ✅ **启动清理**：应用启动时立即清理一次过期日志
- ✅ **手动清理**：可在Web界面手动触发清理操作

### 2. 日志查看功能
- 📋 **实时查看**：在Web界面查看最新的日志记录
- 🔍 **关键词搜索**：支持按关键词搜索日志内容
- 🎯 **级别过滤**：可按ERROR、WARNING、INFO、DEBUG级别筛选
- 📁 **文件选择**：可查看历史日志文件
- 📊 **统计分析**：显示错误、警告、信息的数量统计

### 3. 日志文件管理
- 📂 **文件列表**：显示所有日志文件及其大小、修改时间
- 💾 **空间统计**：显示日志文件总数和占用空间
- 🗑️ **一键清理**：支持手动清理过期日志

## 使用方法

### 访问日志模块

1. 登录Web管理界面（默认：`http://127.0.0.1:5000`）
2. 点击顶部导航栏的 **"📋 日志查看"** 标签
3. 系统会自动加载最新的日志记录

### 查看日志

#### 基本查看
- 默认显示最新的100条日志记录
- 最新的日志显示在顶部
- 不同级别的日志用不同颜色标识：
  - 🔴 **ERROR**：红色，表示错误
  - 🟡 **WARNING**：黄色，表示警告
  - 🔵 **INFO**：蓝色，表示信息
  - ⚫ **DEBUG**：灰色，表示调试信息

#### 过滤日志

1. **按日志文件过滤**
   - 在"日志文件"下拉框中选择要查看的文件
   - 选择"最新日志"查看当前日志

2. **按级别过滤**
   - 在"日志级别"下拉框中选择：ERROR、WARNING、INFO、DEBUG
   - 选择"全部"显示所有级别的日志

3. **按显示行数过滤**
   - 可选择显示50、100、200、500行
   - 默认显示100行

4. **按关键词搜索**
   - 在搜索框中输入关键词
   - 按Enter键或点击"🔍 搜索"按钮
   - 支持中英文搜索

### 刷新日志

点击 **"🔄 刷新"** 按钮可重新加载最新的日志内容。

### 清理过期日志

1. 点击 **"🗑️ 清理过期日志"** 按钮
2. 确认操作
3. 系统会删除7天前的所有日志文件

## 日志内容说明

### 日志格式

每条日志记录包含以下信息：

```
时间戳 - 日志记录器名称 - 级别 - 消息内容
```

示例：
```
2025-11-06 12:34:56 - label_printer - INFO - MQTT客户端已启动
2025-11-06 12:35:01 - label_printer - ERROR - 打印失败：打印机未连接
```

### 日志级别说明

| 级别 | 含义 | 使用场景 |
|------|------|----------|
| ERROR | 错误 | 系统出现错误，功能无法正常执行 |
| WARNING | 警告 | 可能存在问题，但不影响主要功能 |
| INFO | 信息 | 正常的操作信息记录 |
| DEBUG | 调试 | 详细的调试信息 |

## 常见问题排查

### 1. 打印失败问题

**查看ERROR级别日志**：
1. 切换到"📋 日志查看"标签
2. 在"日志级别"中选择"ERROR"
3. 查找与打印相关的错误信息
4. 根据错误提示进行相应处理

常见错误信息：
- `打印机未连接`：检查打印机连接状态
- `无法打开文件`：检查文件路径和权限
- `编码错误`：检查文件编码格式
- `MQTT连接失败`：检查MQTT服务器配置

### 2. MQTT连接问题

**搜索MQTT相关日志**：
1. 在搜索框输入"MQTT"或"mqtt"
2. 点击搜索
3. 查看连接状态和错误信息

### 3. 中文乱码问题

**搜索编码相关日志**：
1. 在搜索框输入"编码"或"encoding"
2. 查看文件编码相关的警告信息
3. 确保文件使用UTF-8编码

## 技术实现

### 后端API

新增以下API接口：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/logs` | GET | 获取日志内容 |
| `/api/logs/files` | GET | 获取日志文件列表 |
| `/api/logs/stats` | GET | 获取日志统计信息 |
| `/api/logs/clean` | POST | 手动清理过期日志 |

### 日志保存位置

日志文件保存在：`data/logs/`

文件命名格式：`{logger_name}_{日期}.log`

示例：`label_printer_20251106.log`

### 自动清理机制

1. **应用启动时**：立即清理一次过期日志
2. **定时任务**：每天凌晨3点自动清理
3. **手动触发**：通过Web界面手动清理
4. **保留策略**：保留最近7天的日志，删除更早的文件

## 日志模块配置

日志管理器默认配置：
- 日志目录：`data/logs`
- 保留天数：7天
- 单个日志文件最大大小：10MB
- 日志备份数量：5个

如需修改配置，可在 `src/utils/log_manager.py` 中调整 `LogManager` 类的初始化参数。

## 最佳实践

### 1. 定期查看日志
- 建议每天查看一次ERROR和WARNING级别的日志
- 及时处理异常情况

### 2. 搜索问题日志
- 遇到问题时，使用关键词搜索相关日志
- 常用关键词：打印、连接、失败、错误、异常

### 3. 保持日志清洁
- 定期清理不需要的日志文件
- 避免日志文件占用过多磁盘空间

### 4. 日志分析
- 关注ERROR数量的变化趋势
- WARNING数量增加可能预示潜在问题
