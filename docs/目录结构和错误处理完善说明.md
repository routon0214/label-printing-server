    """确保所有必要的目录存在"""
    directories = [
        'config',
        'logs',
        'failed_labels'
    ]
    
    for directory in directories:
        try:
            os.makedirs(directory, exist_ok=True)
        except Exception as e:
            print(f"警告：无法创建目录 {directory}: {e}")
```

if __name__ == "__main__":

## 🎯 解决的问题

### 问题1: 目录不存在导致闪退 ❌
**现象**: 程序尝试保存失败的ZPL到 `failed_labels/` 时，因为目录不存在而崩溃

**解决**: 
- ✅ 启动时自动创建所有必要目录
- ✅ 保存文件前再次确认目录存在
- ✅ 即使保存失败也只显示警告，不会崩溃

### 问题2: 导入错误立即闪退 ❌
**现象**: 如果缺少依赖包，导入时直接闪退

**解决**:
- ✅ 最外层捕获所有异常（包括导入错误）
- ✅ 显示友好的错误信息和解决建议
- ✅ 等待用户确认后退出

### 问题3: 编码错误导致异常 ❌
**现象**: Windows 控制台 GBK 编码无法显示 UTF-8 字符

**解决**:
- ✅ 程序启动时自动修复控制台编码
- ✅ 设置 UTF-8 输出
- ✅ 错误时使用 replace 模式而不是崩溃

## 📊 错误处理流程图

```
启动程序
    │
    ├─→ 修复控制台编码
    │       └─→ 失败：静默处理，继续
    │
    ├─→ 导入模块
    │       └─→ 失败：最外层捕获 → 显示错误 → 等待确认 → 退出
    │
    ├─→ 创建目录
    │       └─→ 失败：显示警告，继续
    │
    ├─→ 加载配置
    │       └─→ 失败：显示错误 → 等待确认 → 退出
    │
    ├─→ 启动MQTT客户端
    │       ├─→ 连接失败：显示错误+提示 → 等待确认 → 退出
    │       └─→ 用户中断(Ctrl+C)：立即退出（不等待）

## 🔍 常见问题排查

### Q1: 程序还是闪退了
A: 
1. 确认是双击运行还是命令行运行
2. 尝试用命令行运行查看错误
3. 运行测试脚本检查环境

### Q2: 看不到中文
A: 已自动修复，如果还有问题：
1. 确认控制台编码设置
2. 尝试使用 `chcp 65001` 设置 UTF-8

### Q3: 目录创建失败
A: 
1. 检查文件权限
2. 确认不是在只读目录运行
3. 以管理员身份运行（Windows）

### Q4: 依赖包缺失
A: 
```bash
pip install -r requirements.txt
```
