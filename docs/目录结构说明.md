## 目录说明

### src/
源代码目录，包含所有核心业务代码。

- **config/**: 配置文件目录，存放运行时配置文件（`printer_config.json`等）
- **src/config/**: 配置管理模块，负责配置文件的加载、验证和默认值生成的代码
- **core/**: 核心业务逻辑模块
  - `printer.py`: 打印机控制核心类，支持Windows和Linux平台
  - `mqtt_client.py`: MQTT消息接收和处理，连接MQTT服务器并处理打印任务
  - `zpl_generator.py`: ZPL标签代码生成器，将JSON数据转换为ZPL指令
  - `pdf_printer.py`: PDF文档打印支持
  - `escpos_printer.py`: ESC/POS热敏打印机支持
- **utils/**: 通用工具函数
  - `font_utils.py`: 字体路径查找和加载
  - `image_utils.py`: 文本转图像功能，支持中文
  - `fuzzy_match.py`: 打印机名称模糊匹配算法

### scripts/
脚本目录，包含构建脚本和工具脚本。

- **build.py**: 跨平台构建脚本，使用PyInstaller打包应用
- **build-windows.bat**: Windows平台的构建脚本
- **build-linux.sh**: Linux平台的构建脚本
- **tools/**: 各种工具脚本
  - `fix_paper_issue.bat`: 修复纸张问题脚本
  - `test_exe.bat`: 测试可执行文件脚本

### tests/
测试目录，包含所有测试脚本和测试数据。

### docs/
文档目录，包含项目文档、说明文档、诊断脚本等。

### data/
运行时数据目录，包含程序运行过程中产生的数据。

- **logs/**: 日志文件目录
- **failed_labels/**: 打印失败的标签ZPL代码保存目录

### build/ 和 dist/
构建相关目录，不应提交到版本控制。

- **build/**: PyInstaller构建过程中的临时文件
- **dist/**: 构建输出的可执行文件

## 导入路径规范

所有模块导入应使用 `src.` 前缀：

```python
from src.config import ConfigManager
from src.core import LabelPrintMQTT
from src.utils.fuzzy_match import fuzzy_search_printer
```

## 文件命名规范

- Python文件使用小写字母和下划线：`mqtt_client.py`
- 配置文件使用小写字母和下划线：`printer_config.json`
- 文档文件使用中文或英文，大写开头：`README.md`
- 脚本文件使用平台特定扩展名：`build-windows.bat`, `build-linux.sh`

## 新文件创建指南

1. **源代码文件**: 放在 `src/` 目录下的对应子目录中
2. **测试文件**: 放在 `tests/` 目录中
3. **文档文件**: 放在 `docs/` 目录中
4. **脚本文件**: 放在 `scripts/` 目录中
5. **配置文件**: 放在 `config/` 目录中（项目根目录下的config文件夹）
6. **运行时数据**: 程序会自动创建在 `data/` 目录中

## 注意事项

- 所有导入路径都已更新为使用 `src.` 前缀
- 配置文件路径为 `config/printer_config.json`（在项目根目录的config文件夹下）
- 运行时数据（日志、失败的标签）保存在 `data/` 目录下
- 旧的 `logs/` 和 `failed_labels/` 目录已移动到 `data/` 下
