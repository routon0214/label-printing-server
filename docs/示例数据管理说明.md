# 示例数据管理说明

## 功能概述

新的示例数据管理系统将所有示例数据从文件中动态加载，实现了**数据与代码分离**，便于维护和扩展。

## 系统架构

### 1. 文件结构

```
data/test_samples/
├── examples.json              # 示例配置文件（定义示例列表）
├── escpos_chinese.json        # ESC/POS小票（结构化）
├── escpos_raw_text.json       # ESC/POS小票（纯文本）
├── zd888_chinese_simple.json  # 简单中文标签
├── zd888_chinese_mixed.json   # 中英文混合标签
└── zd888_english.zpl          # 纯英文ZPL
```

### 2. 示例配置文件格式

`data/test_samples/examples.json` 定义了所有可用的示例：

```json
{
  "examples": [
    {
      "id": "simple_chinese",           // 唯一标识符
      "title": "简单中文标签",           // 显示标题
      "desc": "最简单的中文标签示例",    // 描述
      "category": "标签打印",            // 分类
      "file": "zd888_chinese_simple.json" // 数据文件（相对路径）
    },
    {
      "id": "zpl_chinese",
      "title": "ZPL代码含中文",
      "desc": "直接使用ZPL代码打印",
      "category": "标签打印",
      "file": null,                      // 无文件，使用内联数据
      "data": {                          // 内联数据
        "print_type": "label",
        "zpl_code": "^XA^FO50,50^A0N,40,40^FD你好^FS^XZ"
      }
    },
    {
      "id": "zpl_english",
      "title": "纯英文ZPL",
      "desc": "纯英文ZPL代码",
      "category": "标签打印",
      "file": "zd888_english.zpl",      // ZPL文件
      "wrap_zpl": true                   // 自动包装成JSON格式
    }
  ]
}
```

### 3. API接口

#### 获取示例列表

```http
GET /api/examples
```

**响应示例**：
```json
{
  "success": true,
  "examples": [
    {
      "id": "simple_chinese",
      "title": "简单中文标签",
      "desc": "最简单的中文标签示例",
      "category": "标签打印",
      "file": "zd888_chinese_simple.json"
    }
    // ... 更多示例
  ]
}
```

#### 获取指定示例数据

```http
GET /api/examples/{example_id}
```

**响应示例**：
```json
{
  "success": true,
  "example": {
    "id": "simple_chinese",
    "title": "简单中文标签",
    "desc": "最简单的中文标签示例",
    "category": "标签打印"
  },
  "data": {
    "print_type": "label",
    "title": "测试标签",
    "fields": [
      {"label": "产品", "value": "元件A", "font_size": 25}
    ]
  }
}
```

## 添加新示例

### 方法1：从文件加载

1. **创建示例数据文件**

在 `data/test_samples/` 目录下创建JSON或ZPL文件：

```json
// my_example.json
{
  "print_type": "label",
  "title": "我的标签",
  "fields": [
    {"label": "名称", "value": "测试", "font_size": 30}
  ]
}
```

2. **更新配置文件**

编辑 `data/test_samples/examples.json`，添加新示例：

```json
{
  "examples": [
    // ... 现有示例 ...
    {
      "id": "my_example",              // 唯一ID
      "title": "我的示例",              // 显示名称
      "desc": "这是我的自定义示例",     // 描述
      "category": "标签打印",           // 分类
      "file": "my_example.json"         // 文件名
    }
  ]
}
```

### 方法2：使用内联数据

直接在配置文件中定义数据（适合简单示例）：

```json
{
  "id": "inline_example",
  "title": "内联示例",
  "desc": "数据直接定义在配置中",
  "category": "标签打印",
  "file": null,                    // 不使用文件
  "data": {                        // 直接定义数据
    "print_type": "label",
    "zpl_code": "^XA^FO50,50^A0N,30,30^FD测试^FS^XZ"
  }
}
```

## 示例数据格式

### 标签打印（Label）

#### JSON格式（自动生成ZPL）

```json
{
  "print_type": "label",
  "title": "产品标签",
  "fields": [
    {"label": "名称", "value": "电子元件", "font_size": 28},
    {"label": "编号", "value": "SN-001", "font_size": 25}
  ],
  "barcode": "SN001",           // 可选：条形码
  "qrcode": "SN001"             // 可选：二维码
}
```

#### ZPL代码格式

```json
{
  "print_type": "label",
  "zpl_code": "^XA\n^FO50,50^A0N,40,40^FD你好世界^FS\n^XZ"
}
```

或直接使用 `.zpl` 文件（配置中设置 `wrap_zpl: true`）：

```zpl
^XA
^FO50,50^A0N,40,40^FDHello World^FS
^FO50,120^BCN,80,Y,N,N^FD123456^FS
^XZ
```

### ESC/POS打印（小票）

#### 结构化格式

```json
{
  "print_type": "escpos",
  "title": "销售小票",
  "items": [
    {"name": "商品A", "qty": 2, "price": 15.50},
    {"name": "商品B", "qty": 1, "price": 30.00}
  ],
  "total": 61.00,
  "footer": "谢谢惠顾!"
}
```

#### 纯文本格式

```json
{
  "print_type": "escpos",
  "raw_text": "SIZE 60 mm, 40 mm\nGAP 1 mm, 0 mm\nCLS\nTEXT 20,10,\"TSS24.BF2\",0,1,1,\"容器类型\"\nPRINT 1\n",
  "encoding": "gb2312"
}
```

**注意**：
- `\n` 在JSON中表示换行符
- 推荐使用 `gb2312` 或 `gbk` 编码（中文打印机）

## 前端使用

### 自动加载

页面加载时会自动：
1. 调用 `/api/examples` 获取示例列表
2. 填充下拉选择框（按分类分组）
3. 缓存示例数据以提升性能

### 用户操作流程

1. 用户在"快速示例"中选择示例
2. 系统显示示例预览（标题、描述、代码）
3. 用户点击"📥 加载到编辑器"
4. 数据自动填充到JSON输入框
5. 用户可以修改后发送打印

### JavaScript API

```javascript
// 加载示例列表（页面加载时自动调用）
await loadExamplesList();

// 示例选择改变（下拉框onChange事件）
await onExampleChange();

// 加载选中的示例到编辑器
await loadSelectedExample();
```

## 优势

### 1. 数据与代码分离
- ✅ 示例数据存储在JSON文件中，不在JavaScript代码中
- ✅ 修改示例无需改动前端代码
- ✅ 易于版本控制和备份

### 2. 统一管理
- ✅ 所有示例数据在 `data/test_samples/` 目录
- ✅ 配置文件集中定义示例元数据
- ✅ 便于维护和扩展

### 3. 灵活性
- ✅ 支持从文件加载或内联数据
- ✅ 支持JSON和ZPL等多种格式
- ✅ 支持分类和描述

### 4. 性能优化
- ✅ 客户端缓存已加载的示例
- ✅ 按需加载（只在选择时加载数据）
- ✅ 减少初始加载时间

## 故障排查

### 示例不显示

1. 检查 `data/test_samples/examples.json` 是否存在
2. 检查JSON格式是否正确
3. 查看浏览器控制台是否有错误
4. 检查示例文件路径是否正确

### 示例加载失败

1. 确认示例文件存在于 `data/test_samples/` 目录
2. 检查文件编码（应为UTF-8）
3. 查看服务器日志（控制台输出）
4. 验证JSON格式是否正确

### 中文乱码

1. 确保所有文件使用 **UTF-8** 编码保存
2. ESC/POS打印使用 `"encoding": "gb2312"`
3. 检查打印机字符集配置
