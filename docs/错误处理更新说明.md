## 💡 实现细节

### wait_for_user_exit() 函数

```python
def wait_for_user_exit():
    """等待用户按键退出"""
    try:
        input("\n按回车键退出...")
    except:
        pass
```

**特点**:
- ✅ 使用 `try-except` 防止异常
- ✅ 跨平台兼容（Windows/Linux/macOS）
- ✅ 简单明了的提示信息
- ✅ 即使 input 失败也不会崩溃

### 异常处理层级

```python
def main():
    try:
        # 主要逻辑
        try:
            # MQTT客户端
            client.start()
        except KeyboardInterrupt:
            # 用户中断（正常退出，不等待）
            return 0
        except Exception as e:
            # MQTT相关错误（等待确认）
            wait_for_user_exit()
            return 1
    except Exception as e:
        # 外层捕获所有其他错误（等待确认）
        wait_for_user_exit()
        return 1
```

**设计思路**:
1. 内层 try: 处理 MQTT 相关错误
2. 外层 try: 处理所有其他未预期错误
3. KeyboardInterrupt: 正常退出，不等待
4. 其他 Exception: 显示错误并等待

## 📝 注意事项

### 1. 正常退出（Ctrl+C）不会等待
用户按 `Ctrl+C` 主动停止服务时，程序会立即退出，不会要求按回车。

### 2. 后台运行
如果程序在后台运行（如 systemd 服务），`input()` 会被跳过，不会阻塞。

### 3. 日志记录
建议配合日志系统使用，错误信息既显示在屏幕上，也保存到日志文件。
