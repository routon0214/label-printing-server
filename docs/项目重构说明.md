# 项目重构说明

## 📋 重构概述

本项目已从单文件架构重构为模块化工程结构，提高了代码的可维护性和可扩展性。

## 🎯 模块说明

### app.py - 主入口
- 负责启动整个服务
- 加载配置
- 初始化MQTT客户端

### config/ - 配置管理
- **config.py**: 配置文件加载和管理
- **printer_config.json**: 打印机和MQTT配置

### core/ - 核心功能
- **printer.py**: 跨平台打印机控制
  - Windows USB打印
  - Linux CUPS打印
  - 网络打印
  - 模糊搜索功能
- **mqtt_client.py**: MQTT消息接收和处理
- **zpl_generator.py**: ZPL标签代码生成

### utils/ - 工具函数
- **font_utils.py**: 获取系统字体路径
- **image_utils.py**: 文本转图像，支持中文
- **fuzzy_match.py**: 打印机名称模糊匹配算法

### tests/ - 测试模块
- **test_mqtt_send.py**: MQTT消息发送测试
- **test-chinese.py**: 中文打印测试
- **test-zt411.py**: ZT411打印机测试

## 🚀 迁移指南

### 从旧版本迁移

1. **备份配置文件**
   ```bash
   cp printer_config.json printer_config.json.backup
   ```

2. **更新代码**
   ```bash
   git pull
   ```

3. **移动配置文件**（如果需要）
   ```bash
   # 配置文件会自动复制到 config/ 目录
   # 或手动复制
   cp printer_config.json.backup config/printer_config.json
   ```

4. **测试新版本**
   ```bash
   python app.py
   ```

## 🐛 常见问题

### Q: 旧的 zebra_mqtt_printer.py 还能用吗？
A: 可以，但建议使用新的 `app.py` 入口，功能更完善。

### Q: 配置文件需要修改吗？
A: 不需要，格式完全兼容。只需放在 `config/` 目录下。

### Q: 如何回退到旧版本？
A: 
```bash
git checkout <旧版本commit>
python zebra_mqtt_printer.py
```

### Q: 模块导入错误怎么办？
A: 确保在项目根目录运行：
```bash
cd /path/to/label-printing-server
python app.py
```

### Q: 能否同时运行新旧版本？
A: 可以，但建议只运行一个版本以避免端口冲突。

## 📝 开发建议

1. **遵循模块化原则**
   - 每个模块职责单一
   - 避免循环导入

2. **保持向后兼容**
   - 添加新功能时保持旧API可用
   - 使用废弃警告而非直接删除

3. **编写单元测试**
   - 为新功能添加测试
   - 确保测试覆盖率

4. **文档更新**
   - 修改功能时更新文档
   - 添加示例代码
