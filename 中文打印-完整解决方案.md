### 问题根源

1. **控制台乱码** ✅ 已解决
   - 原因: Unicode特殊字符（✓ ✗ ⚠）在Windows控制台显示为乱码

   - 实现: 集成到Web应用和MQTT服务

3. **得力打印机中文乱码** ✅ 已解决
   - 原因: 需要使用GB2312编码
   - 解决: 自动使用GB2312编码和字符集设置
   - 实现: ESCPOSPrinter类自动处理

## 📚 技术细节

### 斑马打印机中文原理

**为什么不能直接打印中文？**
- ZPL标签打印机不支持中文字体
- 即使打印机有中文字库，ZPL命令也无法直接调用

**解决方案**:
- 将中文渲染为黑白位图
- 转换为ZPL ^GFA图像命令
- 打印机接收的是图像数据

**示例**:
```zpl
# 错误 ❌
^FO50,50^A0N,40,40^FD测试^FS

# 正确 ✅
^FO50,50^GFA,432,432,12,000000FFFFFF...^FS
```

### 得力打印机中文原理

**为什么必须用GB2312？**
- ESC/POS打印机基于早期设计
- 内置GB2312字符集
- 不支持UTF-8的多字节序列

**关键命令**:
```
ESC @ (0x1B 0x40)     - 初始化打印机
ESC t 14 (0x1B 0x74 0x0E) - 设置GB2312字符集
```
