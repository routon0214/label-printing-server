# 中文打印乱码 - 完整解决方案

## 问题根源

您的ZPL标签乱码有两个原因：

### 1. **文件编码问题**（已修复✅）
原始文件中的中文被错误编码：
- 原文：`瀹瑰櫒绫诲瀷` (乱码)
- 正确：`容器类型`

### 2. **ZPL中文打印方式问题**（已修复✅）
ZPL标签打印机**不支持**直接打印中文TEXT命令：
```zpl
TEXT 20,10,"TSS24.BF2",0,1,1,"容器类型"  ❌ 错误
```

必须将中文转换为图像：
```zpl
^FO20,10^GFA,432,432,12,00000000...^FS  ✅ 正确
```

## 修复步骤

### 步骤1: 修复文件编码
```bash
python fix_zpl_encoding.py data/print_text.txt data/print_text_fixed.zpl
```

### 步骤2: 格式化换行符
```bash
python -c "content = open('data/print_text_fixed.zpl', 'r', encoding='utf-8').read().replace('\\n', '\n'); open('data/print_text_formatted.zpl', 'w', encoding='utf-8').write(content)"
```

### 步骤3: 转换中文为图像
```bash
python convert_zpl_chinese.py data/print_text_formatted.zpl data/print_text_ready.zpl
```

**或者，使用一键修复脚本**（推荐）：


## 常见问题

### Q1: 转换后的标签位置不对？
A: 原始ZPL使用的坐标系可能与标准ZPL不同，需要调整 `^FO` 命令的坐标。

### Q2: 字体太小或太大？
A: 在 `convert_zpl_chinese.py` 中调整 `font_size` 参数，或在JSON数据中指定 `font_size`。

### Q3: 二维码无法扫描？
A: 检查二维码数据是否正确，尝试增大二维码尺寸。

### Q4: 打印出来是空白？
A: 
1. 检查打印机是否正确连接
2. 使用诊断工具：`python scripts/diagnose_encoding.py`
3. 查看日志：`data/logs/`

### Q5: 新的ZPL文件如何处理？
A: 始终使用转换工具处理包含中文的ZPL文件：
```bash
python fix_zpl_encoding.py your_file.zpl
python convert_zpl_chinese.py your_file_fixed.zpl
```

## 技术原理

### 为什么不能直接打印中文？

1. **ZPL标签打印机限制**:
   - Zebra ZT411等标签打印机不支持中文字体
   - `TSS24.BF2` 字体即使安装也可能不包含所需的中文字符

2. **解决方案**:
   - 将中文文本转换为黑白图像
   - 使用 `^GFA` (Graphic Field Alpha) 命令嵌入图像
   - 图像以十六进制格式编码

### 转换过程

```
中文文本 "容器类型"
    ↓
使用Pillow库渲染为黑白图像 (96x36像素)
    ↓
转换为ZPL十六进制格式
    ↓
^FO20,10^GFA,432,432,12,00000000FFFFFF...^FS
```
