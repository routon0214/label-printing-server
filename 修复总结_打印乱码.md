- **缺少字符集设置命令** (ESC t 14)
- 参考成功案例 `D:\workspaces\py-scripts\print.py` 后确认问题

## ✅ 修复方案

### 1. 文件编码处理优化 (`web_app.py`)

**修复前**:
```python
zpl_code = file_content.decode('utf-8', errors='ignore')  # ❌ 丢失字符
```

**修复后**:
```python
try:
    zpl_code = file_content.decode('utf-8')
except UnicodeDecodeError:
    try:
        zpl_code = file_content.decode('gbk')  # 支持GBK
    except:
        zpl_code = file_content.decode('utf-8', errors='replace')
        print("警告: 文件包含无法解码的字符")
```

### 2. ESC/POS打印机编码修复 (`src/core/escpos_printer.py`) ⭐ **核心修复**

**修复前**:
```python
# 缺少字符集设置
commands.extend(self.CMD_INIT)
commands.extend(text.encode('utf-8'))  # ❌ 错误编码
```

**修复后**:
```python
# 1. 初始化打印机
commands.extend(self.CMD_INIT)

# 2. 设置字符集为简体中文（关键！）
commands.extend(b'\x1B\x74\x0E')  # ESC t 14 = GB2312

# 3. 使用GB2312编码
commands.extend(text.encode('gb2312'))  # ✓ 正确编码
```

## 🎓 技术要点

### ESC/POS打印机为什么必须用GB2312？

1. **硬件设计**: 打印机固件内置GB2312字符集
2. **字符集命令**: ESC t n 设置代码表，n=14 = GB2312
3. **直接映射**: GB2312字节直接映射到打印机字体
4. **UTF-8不兼容**: 打印机不支持多字节UTF-8序列

### ZPL标签打印机的方案

1. **中文转图像**: 使用Pillow + 中文字体
2. **UTF-8文本**: 英文数字使用ZPL内置字体
3. **混合方式**: 中文图像 + 英文文本
