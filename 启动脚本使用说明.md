# 启动脚本使用说明

## 概述

优化后的启动脚本现在支持以下功能：

✅ **自动创建虚拟环境** - 首次运行时自动创建独立的 Python 虚拟环境  
✅ **自动安装依赖** - 检测并自动安装所有必需的依赖包  
✅ **使用阿里云镜像** - 使用阿里云 PyPI 镜像源，大幅提升下载速度  
✅ **自动激活环境** - 每次运行时自动激活虚拟环境  
✅ **升级 pip** - 确保使用最新版本的 pip 工具  

## 使用方法

### Windows 系统

双击运行 `start.bat` 文件，或在命令行中执行：

```cmd
start.bat
```

### Linux/Mac 系统

在终端中执行：

```bash
chmod +x start.sh  # 首次运行需要添加执行权限
./start.sh
```

## 首次运行流程

首次运行启动脚本时，会自动执行以下步骤：

1. ✅ 检查 Python 是否已安装
2. 🔧 配置阿里云 pip 镜像源
3. 📦 创建虚拟环境（`venv` 文件夹）
4. 🔄 激活虚拟环境
5. ⬆️ 升级 pip 到最新版本
6. 📥 安装所有依赖包（从 `requirements.txt`）
7. 🚀 启动打印服务

**首次安装可能需要 2-5 分钟**，取决于您的网络速度。

## 后续运行

第二次及以后运行时，脚本会：

1. ✅ 检测到已存在的虚拟环境，跳过创建步骤
2. 🔄 激活虚拟环境
3. ✅ 检测依赖已安装，跳过安装步骤
4. 🚀 直接启动服务

**后续启动只需几秒钟！**

## 虚拟环境说明

### 什么是虚拟环境？

虚拟环境是一个独立的 Python 运行环境，它的优势包括：

- 🔒 **隔离依赖** - 不会影响系统全局的 Python 包
- 🛡️ **避免冲突** - 不同项目可以使用不同版本的依赖
- 🧹 **易于清理** - 删除 `venv` 文件夹即可完全清除
- 📦 **便于迁移** - 通过 `requirements.txt` 轻松在其他机器上重建

### 虚拟环境位置

虚拟环境创建在项目根目录下的 `venv` 文件夹中：

```
label-printing-server/
├── venv/                    # 虚拟环境（已添加到 .gitignore）
│   ├── Scripts/            # Windows 下的可执行文件
│   ├── bin/                # Linux/Mac 下的可执行文件
│   └── Lib/                # 安装的依赖包
├── start.bat
├── start.sh
└── requirements.txt
```

### 重建虚拟环境

如果遇到问题，可以删除虚拟环境重新创建：

**Windows:**
```cmd
rmdir /s /q venv
start.bat
```

**Linux/Mac:**
```bash
rm -rf venv
./start.sh
```

## 阿里云镜像源配置

脚本已自动配置使用阿里云 PyPI 镜像源：

- 🌐 镜像地址：`https://mirrors.aliyun.com/pypi/simple/`
- ⚡ **下载速度提升 10-50 倍**（相比官方源）
- 🇨🇳 国内服务器，稳定可靠

### 手动配置（可选）

如果需要永久配置 pip 使用阿里云镜像，可以创建配置文件：

**Windows:**

在 `%APPDATA%\pip\pip.ini` 文件中添加：

```ini
[global]
index-url = https://mirrors.aliyun.com/pypi/simple/
trusted-host = mirrors.aliyun.com
```

**Linux/Mac:**

在 `~/.pip/pip.conf` 文件中添加：

```ini
[global]
index-url = https://mirrors.aliyun.com/pypi/simple/
trusted-host = mirrors.aliyun.com
```

## 常见问题

### Q1: 提示"未找到 Python"

**解决方案:**
- 确保已安装 Python 3.7 或更高版本
- Windows: 从 [python.org](https://www.python.org/downloads/) 下载安装
- 安装时勾选 "Add Python to PATH"

### Q2: 虚拟环境创建失败

**解决方案:**
```bash
# 确保 Python 自带的 venv 模块可用
python -m venv --help

# Ubuntu/Debian 系统需要安装
sudo apt install python3-venv
```

### Q3: 依赖安装速度慢

**解决方案:**
- 脚本已配置阿里云镜像，通常很快
- 检查网络连接
- 尝试其他镜像源（清华、豆瓣等）

### Q4: 依赖安装失败

**解决方案:**
```bash
# 删除虚拟环境重新创建
rm -rf venv  # 或 rmdir /s /q venv (Windows)

# 手动安装依赖
python -m venv venv
source venv/bin/activate  # 或 venv\Scripts\activate (Windows)
pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/
```

### Q5: Windows 系统提示"无法加载文件"

**解决方案:**
```powershell
# 以管理员身份运行 PowerShell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```
