# 快速开始指南

欢迎使用 Label Printing Server！本指南将帮助你在 5 分钟内完成项目配置并启动服务。

## 📋 前置要求

在开始之前，请确保已安装：

- ✅ **Python 3.7+** （推荐 Python 3.10+）
  - Windows: [下载安装](https://www.python.org/downloads/)
  - Linux: `sudo apt install python3 python3-pip python3-venv`
  - macOS: `brew install python3`

## 🚀 三步快速启动

### 第一步：初始化项目（仅首次）

#### Windows 系统

双击运行 `init.bat`，或在命令行执行：

```cmd
init.bat
```

#### Linux/Mac 系统

在终端执行：

```bash
chmod +x init.sh
./init.sh
```

初始化过程包括：
- ✅ 创建虚拟环境
- ✅ 安装所有依赖（使用阿里云镜像，速度快！）
- ✅ 创建目录结构
- ✅ 生成配置文件

**首次初始化约需 2-5 分钟，请耐心等待...**

---

### 第二步：配置打印机

编辑配置文件 `config/printer_config.json`：

```json
{
  "mqtt": {
    "url": "mqtt://127.0.0.1:1883",     // MQTT 服务器地址
    "topic": "zebra/print",              // 订阅主题
    "username": null,                    // MQTT 用户名（可选）
    "password": null                     // MQTT 密码（可选）
  },
  "web": {
    "username": "admin",                 // Web 登录用户名
    "password": "admin"                  // Web 登录密码（建议修改）
  },
  "printers": [
    {
      "name": "ZT411",                   // 打印机名称
      "types": ["label"],                // 支持的类型: label, pdf, receipt
      "ip": "192.168.1.100",             // 打印机 IP 地址
      "port": 9100,                      // 打印机端口
      "default": true                    // 是否为默认打印机
    }
  ]
}
```

**主要配置项：**
- `mqtt.url` - MQTT 服务器地址
- `printers[].name` - 打印机名称（支持模糊搜索）
- `printers[].ip` - 打印机 IP 地址
- `printers[].types` - 支持的打印类型

## 📡 使用 MQTT 打印

服务启动后，可以通过 MQTT 发送打印任务：

### 发送 ZPL 标签

```python
import paho.mqtt.client as mqtt
import json

# 连接 MQTT
client = mqtt.Client()
client.connect("127.0.0.1", 1883, 60)

# 发送打印任务
message = {
    "printer": "ZT411",              # 打印机名称（可选，不指定则使用默认）
    "data": "^XA^FO50,50^A0N,50,50^FDHello World^FS^XZ",  # ZPL 内容
    "copies": 1                      # 打印份数（可选）
}

client.publish("zebra/print", json.dumps(message))
client.disconnect()

print("打印任务已发送！")
```

### 发送 Base64 编码的文件

```python
import paho.mqtt.client as mqtt
import json
import base64

# 读取文件并编码
with open("label.pdf", "rb") as f:
    file_data = base64.b64encode(f.read()).decode()

# 连接 MQTT
client = mqtt.Client()
client.connect("127.0.0.1", 1883, 60)

# 发送打印任务
message = {
    "printer": "HP LaserJet",
    "data": file_data,
    "type": "pdf",                   # 文件类型：pdf, image, receipt
    "encoding": "base64"             # 编码方式
}

client.publish("zebra/print", json.dumps(message))
client.disconnect()

print("PDF 打印任务已发送！")
```

## 🔧 故障排除

### 问题 1: 启动后提示"未找到 Python"

**解决方案:**
- 确保已安装 Python 3.7+
- Windows 安装时勾选 "Add Python to PATH"
- 重启命令行窗口

### 问题 2: 打印机连接失败

**检查清单:**
1. ✅ 打印机已开机
2. ✅ 电脑和打印机在同一网络
3. ✅ 打印机 IP 地址配置正确
4. ✅ 打印机端口配置正确（通常是 9100）
5. ✅ 防火墙未阻止连接

**测试命令:**

```bash
# Windows
telnet 192.168.1.100 9100

# Linux/Mac
nc -zv 192.168.1.100 9100
```

### 问题 3: MQTT 连接失败

**检查清单:**
1. ✅ MQTT 服务器已启动
2. ✅ MQTT 服务器地址配置正确
3. ✅ 用户名和密码正确（如有）
4. ✅ 网络连接正常

**运行诊断脚本:**

```bash
# Windows
快速诊断.bat

# Linux/Mac
./快速诊断.sh
```

### 问题 4: 中文打印乱码

**解决方案:**

查看详细的解决方案文档：
- [打印乱码_解决方案.md](打印乱码_解决方案.md)
- [docs/故障排除_打印乱码.md](docs/故障排除_打印乱码.md)

常见原因：
- ❌ 打印机不支持中文字体
- ❌ ZPL 命令编码错误
- ❌ 字体文件未正确加载

### 问题 5: 虚拟环境问题

**完全重建环境:**

```bash
# Windows
rmdir /s /q venv
init.bat

# Linux/Mac
rm -rf venv
./init.sh
```

## 🎯 常见使用场景

### 场景 1: 仓库标签打印

使用斑马打印机打印条码标签：

```python
# 发送 ZPL 命令
zpl = """
^XA
^FO50,50^BY2
^BCN,100,Y,N,N
^FD123456789^FS
^FO50,200^A0N,30,30^FD仓库A-货架01^FS
^XZ
"""

message = {
    "printer": "ZT411",
    "data": zpl
}
client.publish("zebra/print", json.dumps(message))
```

### 场景 2: 小票打印

使用 ESC/POS 打印机打印收据：

```python
message = {
    "printer": "热敏打印机",
    "type": "receipt",
    "data": {
        "title": "购物小票",
        "items": [
            {"name": "商品A", "price": 29.99, "qty": 2},
            {"name": "商品B", "price": 15.50, "qty": 1}
        ],
        "total": 75.48
    }
}
client.publish("zebra/print", json.dumps(message))
```

### 场景 3: PDF 文档打印

打印 PDF 文档：

```python
import base64

with open("document.pdf", "rb") as f:
    pdf_data = base64.b64encode(f.read()).decode()

message = {
    "printer": "HP LaserJet",
    "type": "pdf",
    "data": pdf_data,
    "encoding": "base64",
    "copies": 2
}
client.publish("zebra/print", json.dumps(message))
```
