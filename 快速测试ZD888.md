## 可能的问题和解决方案

### 问题1: 中文显示为空白

**原因**: 图像没有正确生成或发送

**检查步骤**:
1. 查看生成的ZPL文件:
```bash
notepad data/zd888_chinese_test.zpl
```

2. 确认文件中包含 `^GFA` 命令（图像数据）
   - 如果有 `^GFA` → 图像生成成功，问题在打印机接收
   - 如果没有 `^GFA` → 图像生成失败

### 问题2: 中文显示为方块

**原因**: 字体问题

**解决**:
```bash
# 检查字体
dir C:\Windows\Fonts\msyh.*

# 如果没有，安装Pillow
pip install Pillow
```

### 问题3: ZPL代码太大打印机无响应

**原因**: ZD888内存可能有限制

**解决**: 减小字体大小或简化内容
```python
# 修改字体大小为更小的值
{"label": "测试", "value": "内容", "font_size": 20}  # 从28改为20
```

### 问题4: 英文正常但中文完全不显示

**原因**: 可能是图像命令格式问题

**测试方法**: 使用网络打印绕过驱动
```bash
python 诊断打印问题.py
# 选择 3 - 网络打印
# 输入ZD888的IP地址
```

## 关键检查点

### ✅ 确认中文转图像功能工作
运行测试时，应该看到类似输出:
```
生成: 标题
使用字体: C:/Windows/Fonts/msyh.ttc
文本 '测试' 尺寸: 60x30
```

如果看到"警告：未找到中文字体"，需要:
```bash
pip install Pillow
```

### ✅ 确认ZPL代码包含图像
生成的ZPL文件应该包含:
```zpl
^XA
^FO50,40^GFA,1234,1234,20,0000FF00...^FS
^XZ
```

`^GFA` 后面跟着长串十六进制数据就是图像。

### ✅ 确认打印机接收
如果前两项都正常但还是不打印:
1. 检查打印机队列是否卡住
2. 尝试重启打印机
3. 尝试减小标签内容

## 技术细节

### ZD888 vs ZT411
- ZD888: 桌面型，203dpi，较小内存
- ZT411: 工业型，300dpi，大内存

**关键差异**: ZD888可能对大ZPL文件更敏感！

### 优化建议
如果ZD888处理大ZPL有问题:

1. **减小字体**:
```python
font_size=20  # 而不是30或40
```

2. **减少内容**:
```python
# 只包含必要的字段
data = {
    "title": "测试",
    "fields": [
        {"label": "名称", "value": "产品", "font_size": 22}
    ]
}
```

3. **压缩图像** (如果需要):
使用更小的padding值
