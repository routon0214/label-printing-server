# 打包指南

## 问题描述

打包后运行时出现错误：`jinja2 must be installed to use Jinja2Templates`

这是因为 PyInstaller 打包时没有正确包含 FastAPI 和 Jinja2 的依赖。

---

## 解决方案

已更新打包脚本 `scripts/build.py`，添加了所有必要的依赖。

### 新增的依赖包含

#### 1. FastAPI 和 Starlette
```python
--collect-all=fastapi
--collect-all=starlette
--copy-metadata=fastapi
--copy-metadata=starlette
--hidden-import=fastapi
--hidden-import=fastapi.responses
--hidden-import=fastapi.staticfiles
--hidden-import=fastapi.templating
--hidden-import=starlette
...（更多子模块）
```

#### 2. Jinja2 模板引擎
```python
--collect-all=jinja2
--copy-metadata=jinja2
--hidden-import=jinja2
--hidden-import=jinja2.ext
```

#### 3. Uvicorn ASGI 服务器
```python
--collect-all=uvicorn
--copy-metadata=uvicorn
--hidden-import=uvicorn
--hidden-import=uvicorn.logging
--hidden-import=uvicorn.loops.auto
--hidden-import=uvicorn.protocols.http.auto
...（更多子模块）
```

#### 4. Pydantic（FastAPI 依赖）
```python
--hidden-import=pydantic
--hidden-import=pydantic.fields
--hidden-import=pydantic.main
```

#### 5. 其他必要依赖
```python
--hidden-import=anyio
--hidden-import=h11
--hidden-import=httptools
--hidden-import=websockets
```

#### 6. 模板和静态文件
```python
--add-data=templates;templates  # Windows
--add-data=templates:templates  # Linux/Mac
--add-data=static;static        # Windows
--add-data=static:static        # Linux/Mac
```

## 常见问题

### 问题1：打包后缺少模板文件

**错误信息：**
```
TemplateNotFound: index.html
```

**解决方法：**
确保打包脚本包含了模板目录：
```python
--add-data=templates{path_sep}templates
```

检查 `dist/` 目录下是否有 `templates/` 文件夹。

### 问题2：打包后找不到静态文件

**错误信息：**
```
StaticFiles directory 'static' does not exist
```

**解决方法：**
1. 确保项目根目录有 `static/` 文件夹
2. 或者在代码中处理静态文件不存在的情况

### 问题3：Windows Defender 误报

打包后的 `.exe` 文件可能被 Windows Defender 标记为病毒。

**解决方法：**
1. 添加到 Windows Defender 排除列表
2. 或使用代码签名证书签名 `.exe` 文件

### 问题4：打包后文件过大

如果打包后文件超过 100MB，可以考虑：

1. **使用目录模式打包**（而不是单文件）
   ```python
   # 修改 build.py，注释掉 --onefile
   # '--onefile',
   ```

2. **排除不需要的依赖**
   ```python
   '--exclude-module=matplotlib'
   '--exclude-module=numpy'
   ```

### 问题5：Linux/Mac 上权限问题

**错误信息：**
```
Permission denied: ./label-printing-server
```

**解决方法：**
```bash
chmod +x label-printing-server
```

## 技术说明

### PyInstaller 工作原理

PyInstaller 打包过程：
1. 分析 `app.py` 的所有导入
2. 收集所有依赖的 Python 包
3. 复制 Python 解释器和依赖库
4. 将所有内容打包成单个文件或目录
5. 添加引导程序

### 为什么需要显式指定依赖

某些库使用动态导入，PyInstaller 无法自动检测：
- FastAPI 使用 `__import__()` 动态加载模块
- Jinja2 的扩展系统
- Uvicorn 的协议自动选择

因此需要使用 `--hidden-import` 显式告诉 PyInstaller。

### --collect-all vs --hidden-import

- `--collect-all=package`：收集包的所有内容（推荐）
- `--hidden-import=module`：只包含特定模块（精确控制）
- `--copy-metadata=package`：复制包的元数据（某些包需要）
