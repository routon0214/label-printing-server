# 打包错误修复

## 错误信息

```
importlib.metadata.PackageNotFoundError: No package metadata was found for jinja2
```

## 问题原因

`--copy-metadata=jinja2` 无法找到包元数据，因为：
1. Jinja2 的实际包名是 `Jinja2`（首字母大写）
2. 导入时使用 `jinja2`（小写）
3. `--copy-metadata` 需要使用包的实际安装名称

## 解决方案

已更新 `scripts/build.py`，移除了有问题的 `--copy-metadata=jinja2`。

### 修改内容

#### 1. 移除 Jinja2 的元数据复制
```python
# 之前：
'--copy-metadata=jinja2',  # ❌ 会导致错误

# 修改后：
# '--copy-metadata=Jinja2',  # ✓ 已移除
```

保留 `--collect-all=jinja2` 就足够了，它会包含所有必要的文件。

#### 2. 增强数据文件检查
```python
print("\n检查数据文件...")
if os.path.exists('templates'):
    cmd.append(f'--add-data=templates{path_sep}templates')
    print("  ✓ 将包含 templates 目录")
else:
    print("  ⚠ templates 目录不存在，跳过")
```

#### 3. 自动清理旧的 spec 文件
```python
# 清理旧的 spec 文件
spec_files = ['label-printing-server.spec', 'app.spec']
for spec_file in spec_files:
    if os.path.exists(spec_file):
        print(f"清理旧的spec文件: {spec_file}")
        os.remove(spec_file)
```

## 其他可能的元数据问题

如果其他包也出现类似问题，可以使用以下方法排查：

### 检查包的实际名称

```python
# 在 Python 中运行
import importlib.metadata
print(importlib.metadata.distribution('fastapi').name)  # 输出: fastapi
print(importlib.metadata.distribution('starlette').name)  # 输出: starlette
```

### 安全的做法

对于大部分包，`--collect-all=package` 就足够了，不需要 `--copy-metadata`。

只有当运行时需要检查包版本时才需要 `--copy-metadata`。

## 常见错误模式

### 错误1：包名大小写不匹配

```
❌ --copy-metadata=jinja2        # 错误：小写
✓  --copy-metadata=Jinja2        # 正确：首字母大写
⭕ 或直接移除，使用 --collect-all
```

### 错误2：包名与导入名不同

```python
# 安装的包名
pip install python-dateutil

# 导入名
import dateutil

# PyInstaller 配置
--collect-all=dateutil           # ✓ 使用导入名
--copy-metadata=python-dateutil  # ✓ 使用包名
```

### 错误3：过度依赖 copy-metadata

```python
# 不推荐：为每个包都添加 copy-metadata
--copy-metadata=package1
--copy-metadata=package2
--copy-metadata=package3

# 推荐：只使用 collect-all
--collect-all=package1
--collect-all=package2
--collect-all=package3
```

## 最佳实践

1. **优先使用 `--collect-all`**
   - 包含包的所有内容
   - 更可靠，更少出错

2. **谨慎使用 `--copy-metadata`**
   - 只在确实需要时使用
   - 确保包名正确

3. **测试打包结果**
   - 每次修改后都要测试
   - 在干净的环境中测试

4. **保留错误日志**
   - 记录打包错误信息
   - 便于排查和解决
