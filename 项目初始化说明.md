# 项目初始化说明

## 概述

`init.py` 是 Label Printing Server 的项目初始化脚本，用于自动化配置开发和运行环境。

## 功能特性

✅ **自动环境检查** - 检查 Python 版本是否满足要求  
✅ **虚拟环境创建** - 自动创建独立的 Python 虚拟环境  
✅ **依赖自动安装** - 使用阿里云镜像快速安装所有依赖  
✅ **目录结构创建** - 自动创建必要的目录结构  
✅ **配置文件生成** - 自动生成默认配置文件  
✅ **安装验证** - 验证所有依赖是否正确安装  

## 使用方法

### 方式一：使用便捷脚本（推荐）

#### Windows 系统

双击运行 `init.bat` 或在命令行中执行：

```cmd
init.bat
```

#### Linux/Mac 系统

在终端中执行：

```bash
chmod +x init.sh  # 首次运行需要添加执行权限
./init.sh
```

### 方式二：直接运行 Python 脚本

```bash
# Windows
python init.py

# Linux/Mac
python3 init.py
```

## 初始化流程

初始化脚本会按以下步骤执行：

### 步骤 1/7: 检查 Python 版本

- 检查 Python 版本是否 ≥ 3.7
- 显示当前版本信息

```
当前 Python 版本: 3.10.5
最低要求版本: 3.7
✓ Python 版本检查通过
```

### 步骤 2/7: 创建虚拟环境

- 在项目根目录创建 `venv` 文件夹
- 如果已存在，询问是否重新创建

```
正在创建虚拟环境: D:\workspaces\label-printing-server\venv
✓ 虚拟环境创建成功
```

### 步骤 3/7: 升级 pip

- 升级虚拟环境中的 pip 到最新版本
- 使用阿里云镜像加速下载

```
正在升级 pip（使用阿里云镜像）...
✓ pip 升级成功
```

### 步骤 4/7: 安装项目依赖

- 从 `requirements.txt` 读取依赖列表
- 使用阿里云镜像安装所有依赖
- **此步骤可能需要 2-5 分钟**

```
正在安装依赖（使用阿里云镜像）...
这可能需要几分钟，请耐心等待...

Collecting paho-mqtt>=1.6.1
Collecting Pillow>=9.0.0
...
✓ 依赖安装完成
```

### 步骤 5/7: 创建目录结构

- 创建以下目录（如果不存在）：
  - `data/` - 数据目录
  - `data/logs/` - 日志目录
  - `data/uploads/` - 上传文件目录
  - `data/failed_labels/` - 失败标签目录
  - `config/` - 配置文件目录
  - `static/` - 静态文件目录
  - `templates/` - 模板文件目录

```
  创建目录: data
  创建目录: data/logs
  创建目录: data/uploads
  ...
✓ 目录结构创建完成
```

### 步骤 6/7: 创建配置文件

- 如果 `config/printer_config.json` 不存在，则创建
- 优先从 `printer_config_example.json` 复制
- 如果示例文件不存在，创建默认配置

```
从示例文件创建配置: printer_config_example.json
✓ 配置文件创建成功: printer_config.json

⚠ 请根据实际情况修改配置文件:
  - MQTT 连接信息
  - Web 界面登录密码
  - 打印机配置
  配置文件位置: config\printer_config.json
```

### 步骤 7/7: 验证安装

- 验证所有关键依赖是否可以正常导入
- 检查的包：
  - `paho.mqtt.client` - MQTT 客户端
  - `PIL` - 图像处理（Pillow）
  - `fastapi` - Web 框架
  - `uvicorn` - ASGI 服务器
  - `jinja2` - 模板引擎

```
  ✓ paho.mqtt.client
  ✓ PIL
  ✓ fastapi
  ✓ uvicorn
  ✓ jinja2
✓ 所有依赖验证通过
```

## 常见问题

### Q1: 提示"未找到 Python"

**解决方案:**

- **Windows:** 
  1. 从 [python.org](https://www.python.org/downloads/) 下载安装
  2. 安装时务必勾选 "Add Python to PATH"
  3. 重启命令行窗口

- **Linux/Mac:**
  ```bash
  # Ubuntu/Debian
  sudo apt update
  sudo apt install python3 python3-pip python3-venv
  
  # CentOS/RHEL
  sudo yum install python3 python3-pip
  
  # macOS
  brew install python3
  ```

### Q2: Python 版本过低

**解决方案:**

项目需要 Python 3.7 或更高版本，请：
1. 卸载旧版本 Python
2. 安装新版本（建议 Python 3.10+）
3. 重新运行初始化脚本

### Q3: 虚拟环境创建失败

**Ubuntu/Debian 系统常见问题:**

```bash
# 安装 venv 模块
sudo apt install python3-venv
```

**Windows 系统常见问题:**

- 确保有写入权限
- 尝试以管理员身份运行
- 检查磁盘空间是否充足

### Q4: 依赖安装失败

**网络问题:**

如果阿里云镜像访问慢，可以尝试其他镜像：

```bash
# 清华镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 豆瓣镜像
pip install -r requirements.txt -i https://pypi.douban.com/simple
```

**权限问题:**

- Linux: 不要使用 `sudo`，在虚拟环境中不需要
- Windows: 尝试以管理员身份运行

**依赖冲突:**

```bash
# 删除虚拟环境重新创建
rm -rf venv  # Linux/Mac
rmdir /s /q venv  # Windows

# 重新运行初始化
python init.py
```

### Q5: 想重新初始化

如果需要完全重新初始化项目：

```bash
# 1. 删除虚拟环境
rm -rf venv  # Linux/Mac
rmdir /s /q venv  # Windows

# 2. 删除配置文件（可选）
rm config/printer_config.json  # Linux/Mac
del config\printer_config.json  # Windows

# 3. 重新运行初始化
python init.py
```

### Q6: 初始化后如何更新依赖

如果项目依赖有更新：

```bash
# 激活虚拟环境后
# Windows
venv\Scripts\activate.bat
pip install -r requirements.txt --upgrade -i https://mirrors.aliyun.com/pypi/simple/

# Linux/Mac
source venv/bin/activate
pip install -r requirements.txt --upgrade -i https://mirrors.aliyun.com/pypi/simple/
```

或者直接运行启动脚本，它会自动检查并安装缺失的依赖。

## 技术细节

### 虚拟环境结构

```
venv/
├── Scripts/          # Windows 可执行文件
│   ├── activate.bat  # 激活脚本
│   ├── python.exe    # Python 解释器
│   └── pip.exe       # pip 包管理器
├── bin/              # Linux/Mac 可执行文件
│   ├── activate      # 激活脚本
│   ├── python        # Python 解释器
│   └── pip           # pip 包管理器
└── Lib/              # 安装的依赖包
    └── site-packages/
```

### 阿里云镜像配置

脚本自动使用阿里云 PyPI 镜像源：

- **镜像地址:** `https://mirrors.aliyun.com/pypi/simple/`
- **优势:** 国内访问速度快，稳定可靠
- **速度提升:** 相比官方源提升 10-50 倍

### 与 start.bat/start.sh 的区别

| 特性 | init.py | start.bat/start.sh |
|------|---------|-------------------|
| **用途** | 项目初始化 | 日常启动服务 |
| **运行频率** | 仅运行一次 | 每次启动都运行 |
| **创建虚拟环境** | ✅ 是 | ✅ 是（如不存在） |
| **安装依赖** | ✅ 完整安装 | ✅ 检查并安装 |
| **创建目录** | ✅ 是 | ❌ 否 |
| **生成配置** | ✅ 是 | ❌ 否 |
| **验证安装** | ✅ 详细验证 | ✅ 快速检查 |
| **启动服务** | ❌ 否 | ✅ 是 |

**推荐使用流程:**

1. **首次设置:** 运行 `init.py` 或 `init.bat/init.sh`
2. **日常使用:** 运行 `start.bat` 或 `start.sh`
