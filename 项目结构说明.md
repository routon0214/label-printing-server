## 🎯 使用场景

### 1. 开发环境运行

```bash
python app.py
```

需要的文件：
- `app.py`
- `config/`
- `core/`
- `utils/`
- `config/printer_config.json`

### 2. 打包发布

```bash
python build.py
```

需要的文件：
- 所有核心文件
- `app.spec`
- `build.py`
- `requirements.txt`

### 3. 生产部署（打包后）

只需要 `dist/` 目录下的内容：
- `label-printing-server.exe` (Windows)
- `label-printing-server/` (Linux)
- `config/`

## 💡 最佳实践

### 代码组织

1. **单一职责**: 每个模块只负责一个功能
2. **松耦合**: 模块间通过接口通信
3. **高内聚**: 相关功能放在同一模块

### 文件命名

1. **模块**: 小写+下划线 `printer_config.py`
2. **类**: 大驼峰 `ZebraPrinter`
3. **函数**: 小写+下划线 `load_config()`
4. **常量**: 大写+下划线 `DEFAULT_PORT`

### 目录规则

1. **模块目录**: 必须包含 `__init__.py`
2. **配置目录**: 只放配置文件
3. **测试目录**: 对应源代码结构
4. **文档目录**: 按主题组织
